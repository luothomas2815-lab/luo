# Agent 协作说明（睡眠 App MVP）

本仓库面向 **数字化 CBT-I + AI 教练** 的 wellness 场景 MVP。人机协作时请遵守本文与 `.cursor/rules/product.mdc`。

## 技术栈（约定）

- **框架**：Next.js（App Router）+ TypeScript  
- **样式**：Tailwind CSS  
- **数据与鉴权**：Supabase  
- **AI**：OpenAI **Responses API**（解释、答疑；**不**替代规则引擎做治疗决策）  
- **部署**：Vercel  

## 架构原则

1. **规则引擎**产出睡眠计划与结构化建议；**LLM** 仅用于自然语言解释与答疑，不得成为唯一决策源。  
2. **合规与安全**：不提供药物建议、不替代医学诊断、不承诺疗效；高风险内容必须触发安全提示与转诊建议（见产品规则）。  
3. **迭代方式**：一次只完成一个边界清晰的任务；合并前至少通过 **lint / typecheck / build / tests** 中一种可自动化验证。  
4. **改动范围**：默认最小化 diff，不顺带大重构或无关文件修改，除非任务明确要求。

## 对 Agent 的期望

- 实现前先阅读相关现有代码与规则文件。  
- 新功能优先复用已有抽象与目录约定；无约定时按 Next.js / React 常见实践。  
- 涉及用户健康与 AI 文案时，默认偏保守、可审计（日志与提示词边界清晰）。  

## 与产品负责人的协作

- **未经确认不要批量实现业务功能**；大步骤按「实施计划」拆任务并逐步验收。  
- 若需求与合规冲突，先标出冲突点再改代码。
